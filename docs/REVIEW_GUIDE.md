# Review Guide

## Review layers

1. Data validation
2. Code execution validation
3. Pedagogical review
4. Accessibility review
5. Copyright/source review
6. Release review

## Pedagogical review checklist

- Is the lesson appropriate for the target stage?
- Are prerequisites explicit?
- Does each exercise match a learning objective?
- Are common mistakes realistic?
- Is the difficulty progression smooth?

## Content accuracy checklist

- Are definitions correct?
- Are answers correct?
- Are edge cases handled?
- Are claims sourced when needed?

For AI-assisted student-facing or teacher-facing prose review, use the
canonical checklist section in `docs/review/MVP_REVIEW_CHECKLIST.md`.

## Release checklist

- Status is `approved` before `published`.
- Changelog updated.
- Data revisions updated.
- Human reviewer noted.
