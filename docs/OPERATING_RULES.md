# Operating Rules

## Rule 1: Human accountability

AI can draft, validate, and review, but humans remain accountable for publication.

## Rule 2: No official-textbook claim

Do not describe this project as an official approved textbook.
Use terms such as open educational material, supplemental resource, or self-study material.

## Rule 3: No unreviewed public release

A record cannot move to `published` without human review.

## Rule 4: Source discipline

Use primary sources where possible.
Do not invent citations.
If a fact needs a source and no source is available, mark it as needing review.

## Rule 5: Copyright safety

Do not copy protected textbook explanations, paid problem-book questions, proprietary diagrams, teacher manuals, or answer keys.
Use original wording and original exercises.
Use quotations only when legally justified and clearly attributed.

## Rule 6: Versioned truth

Answers, rubrics, explanations, and policies may change.
Do not overwrite history silently.
Use revisions and supersession fields.

## Rule 7: Machine checks first

Run validation before requesting human review.
Machine validation does not replace human review.

## Rule 8: Accessibility by default

Use alt text for images.
Do not rely on color alone.
Avoid examples that depend on wealth, region, family structure, or cultural knowledge unless educationally necessary.

## Rule 9: Minimal agent context

Repository files should be concise, structured, and English-first.
Do not dump long private reasoning into docs.
Use short public summaries and decision records.

## Rule 10: Small PRs

One PR should change one coherent unit.
Avoid broad rewrites unless explicitly approved.
