# Release Policy

## Status transition

```text
draft -> machine_checked -> human_review_requested -> approved -> published
```

## Release requirements

- Validation passes.
- Human review complete.
- Changelog updated.
- No unresolved copyright flags.
- No unresolved source flags.
- No unresolved accessibility blockers.

## Versioning

Use semantic versioning for repository releases.
Use record-level version suffixes for educational data.
