## Purpose

Define the checklist for a future v0.1 MVP release of the open educational material.

## Acceptance criteria

- [ ] Required files and records are listed (lessons, teacher guides, problems, answers, rubrics, sources, revisions per `docs/PROCESS.md` definition of done).
- [ ] Machine-check evidence is required (validation and index build output attached).
- [ ] Human review gates are required (pedagogy, copyright, accessibility, release approval).
- [ ] Copyright and source notes are required in release artifacts.
- [ ] Release notes requirements are listed (scope, known limitations, non-official-textbook disclaimer).
- [ ] `published` status cannot be set without human approval.
