# 情報デザイン・プロジェクト

## 学習目標

1. 示された受け手と目的に向けて、独自の情報成果物を設計できる。（B7.O1）
2. 提供された評価証拠を分析し、対話型の修正版でフォーカス移動、可視フォーカス、起動、完了を実際に点検できる。（B7.O2）
3. 証拠を使って、修正内容と残る限界を説明できる。（B7.O3）

## 前提知識

- B6で学んだ、低コストのプロトタイプ、作業中心の評価、観察と推測の区別。

## 準備チェックと復習

B1〜B6から、受け手と目的、デジタル表現、障壁と要件、情報構造、図表、プロトタイプ評価を一文ずつ思い出します。
B6の「作業・成功基準・観察・改訂・再確認」を含む五つ以上を説明できたら先へ進みます。
B6を説明できない場合は必ずB6の到達チェックへ戻ります。
ほかで迷ったら、該当する元の到達チェックへ戻り、短い例を一つ作ってから再挑戦します。

## 考え方のモデル

プロジェクトは「要件→出所確認→構造案→P1評価→P2制作→作業中心の確認→証拠に基づく改訂→限界と再確認」の必須マイルストーンで進めます。各変更を要件と証拠番号へ結び付け、確認していない利用者や環境へ一般化しません。

## 一緒に試す

提供P1の「日時が後ろ」「申込操作が画像だけ」という二点について、対応する要件、観察できる失敗、P2での変更、再確認手順を四列にします。原因を断定せず、事実と追加確認が必要な仮説を分けます。

## 導入

このプロジェクトでは、「しおかぜ地区防災学習会」の参加案内を完成させます。
提供された初版P1と架空の観察ログを分析し、要件と証拠に対応する独自の修正版P2を作ります。
授業の中核活動では、実在する参加者や個人データを使いません。

## 成果物の要件

一画面またはA4一枚相当の案内を作ります。
内容と論理順序の設計には、紙、文章、スライド形式など、特定の製品に依存しない方法を使えます。
ただし、キーボード操作可という結果を提出する場合は、入力欄やボタンを実際に操作できる対話型P2も用意します。

| 項目 | 指定内容 |
| --- | --- |
| 受け手 | 異なる機器、入力方法、読み方、利用状況をもつ架空の地域参加者 |
| 目的 | 日時、場所、持ち物、申込期限を取得し、申込みを完了できるようにする |
| 日時 | 7月27日 15:00 |
| 場所 | しおかぜ交流館 |
| 持ち物 | 筆記用具 |
| 申込期限 | 7月20日 |
| 経路 | 東入口と青い案内板を目印として示す |
| 必須要件 | 意味順の見出し、経路図のテキスト代替、音声の文字起こし、色に頼らない状態表示、キーボードだけで完了できる申込順序 |

## 提供された初版P1

P1は、架空の作成者が用意した固定の初版です。学習者自身が作った成果物ではありません。

- 「開催概要」に日時と場所がある。
- 持ち物の音声と、同じ内容の文字起こしがある。
- 経路図には東入口と青い案内板が描かれているが、本文には入口名と目印がない。
- 申込欄には入力、確認、送信があるが、Tab順では確認から送信へ進めない。
- 申込状態は文字と記号で示されている。

## P1について提供された観察ログ

同じ作業を行った架空の実行結果です。個人を識別するデータではありません。

| 証拠番号 | 提供された観察 |
| --- | --- |
| E1 | 日時を答える作業は3件すべて成功した |
| E2 | キーボードだけの申込作業は2件で確認から送信へ進まず停止した |
| E3 | 音を使わない持ち物確認は3件すべて文字起こしから正答した |
| E4 | 経路本文から入口名を答える作業は2件で失敗した |

このログから、P1の日時配置と文字起こしは今回の作業を支えたと観察できます。
一方、キーボード申込と経路本文は要件を満たしていません。
失敗の唯一の原因や、別の機器・言語・状況での結果までは断定できません。

## 完成させる成果物

提出物は次の四点です。

1. 評価表: E1〜E4を要件別に整理し、観察、未確認の原因、達成・未達を分ける。
2. 修正版P2: 成果物の要件と観察ログに対応する、独自の完成案内。
3. P2確認記録: 完成したP2を指定の方法で実際に点検し、使用環境、操作、観察結果、追加修正を記録する。
4. 改訂説明: 変更点、対応する証拠番号、維持した点、確認結果、残る限界を示す。

## 例題: 小さな別例で証拠を修正へつなぐ

次は、提出課題とは異なる「みなと学習室」の架空例です。
提出課題のP1やE1〜E4への解答ではなく、観察を具体的な修正へつなぐ方法だけを示します。

初版では、部屋の変更を青い帯だけで示していました。
架空ログM1では、3件中2件が変更後の部屋を答えられませんでした。

| 要件 | 観察 | 判断 | 修正版での対応 |
| --- | --- | --- | --- |
| 色を使わず変更後の部屋を取得できる | M1で3件中2件が失敗 | 未達。原因は未断定 | 「会場変更」の見出しと部屋名を文字で加える |

修正版の該当部分は、例えば次のようにします。

> ### 会場変更
>
> 会場は**2階B室**へ変更されました。

改訂説明には「M1に対応して、色だけの帯から、見出しと部屋名を含む文字表示へ変えた」と書けます。
ただし、この小例だけでは、修正版で部屋を取得できるか、別の状況でも同じ結果になるかは分かりません。

## 完成したP2を確認する

P2を完成させたら、成果物だけを使って次の四つを実行します。紙や文字カードで指を使ってたどれるのは、情報の意味順序や、設計した論理順序までです。フォーカスが実際に移動すること、フォーカスが見えること、操作できることの証拠にはしません。

1. 文字だけを読み、日時、場所、持ち物、期限、経路を取得する。
2. 色を隠すか白黒表示にし、申込状態を区別する。
3. 対話型P2をキーボードだけで操作し、次の手順を実行する。
   - ポインティング機器を使わず、開始位置から`Tab`で入力、確認、送信へ進む。
   - 各位置でフォーカスが目で確認でき、今どこにいるかを記録する。
   - `Shift+Tab`で一つ以上前へ戻り、逆方向にも移動できることを確認する。
   - 入力を行い、ボタンなどを`Enter`または`Space`の適切な方で起動する。
   - 送信後に完了メッセージまたは完了状態へ到達し、内容を確認する。
4. 音声を使わず、文字起こしから持ち物を取得する。

キーボード確認には、ブラウザやアプリの名称、機器、確認日、開始状態、押したキー、フォーカスが見えた要素、起動結果、完了結果を記録します。各項目について、実際の結果を「達成」「未達」と観察メモで記録します。
未達があればP2を修正し、どこを直したかも残します。
この確認は一つの環境で成果物の要件を点検するものです。実在する参加者による評価、別の機器や支援技術での結果、WCAG適合、あらゆる状況での成功を証明するものではありません。

## よくある間違い

- P1を学習者自身の成果物として扱う。P1は、E1〜E4と対応する提供済みの固定資料です。
- 「後で評価する」と書き、評価表やP2を完成させない。四つの提出物を実際に作ります。
- 観察ログにない利用者の気持ちや能力を推測する。記録された作業結果だけを証拠にします。
- 見た目だけを変え、失敗した作業との対応を示さない。変更ごとに証拠番号を付けます。
- 紙やカードで順序をたどれたため、キーボード操作できると判断する。紙で確認できるのは情報または論理順序であり、実際のフォーカス、起動、完了は対話型P2で確認します。
- `Tab`で前へ進めただけで操作可能と判断する。逆方向、可視フォーカス、起動、完了も記録します。
- P2の自己点検だけで、すべての受け手や状況で成功すると断定する。確認方法と範囲を示します。

## 自己確認

1. P2だけを見て、日時、場所、持ち物、期限、経路を文字で確認できますか。
2. 各修正と維持点をE1〜E4の観察へたどれますか。
3. 対話型P2で`Tab`、`Shift+Tab`、可視フォーカス、起動、完了を実際に確認し、環境と結果を記録しましたか。
4. 評価表、独自のP2、P2確認記録、改訂説明と限界の四点を提出できますか。

## 自己確認の診断と修正

確認できた状態は、提供P1とログの対応を保ち、要件を満たす独自のP2を完成させ、
指定の四項目を実際に点検し、キーボード確認では順方向・逆方向の移動、可視フォーカス、起動、完了を記録し、証拠番号で修正と維持を説明して、確認範囲と一般化の限界を示していることです。

- P1とログが対応しない場合は、提供P1の五つの特徴へ戻ります。
- 計画だけの場合は、指定内容を実際に配置したP2を作ります。
- P2の確認記録がない場合は、文字、色、対話型キーボード操作、音声なしの四項目を実行して結果を残します。
- 紙の操作順だけの場合は、その結果を論理順序の確認と書き、対話型P2でキーボード確認を追加します。
- 改訂理由が「見やすくした」の場合は、対応する作業、証拠番号、具体的変更を書きます。
- ログにない原因を断定した場合は、観察と追加確認が必要な仮説に分けます。
- 限界がない場合は、実在参加者では未確認であること、架空実行数、機器、言語、自己点検から一般化できる範囲を点検します。

## まとめ

- 提供P1と対応する架空ログを、要件別に評価します。
- 証拠に対応する独自のP2を完成させます。
- 完成したP2を指定の方法で確認し、対話型P2の実キーボード結果と追加修正を記録します。
- 変更と維持の理由、確認範囲、残る限界を説明します。

## 別の場面への転移

完成した成果物を、異なる受け手や利用状況へ作り替えるとします。変える要件と変えない要件を分け、紙面またはカードで新しい構成を示し、どの確認結果を再利用でき、何を試し直す必要があるか説明してください。

これは必須3時限の外で行う25分の自学課題です。20分でA4判1枚の変更要件、維持要件、新構成、再利用できる証拠、再確認する作業を作ります。続く5分で全変更が要件と証拠へたどれるか、残る限界を示したかを照合し、一箇所を修正します。
