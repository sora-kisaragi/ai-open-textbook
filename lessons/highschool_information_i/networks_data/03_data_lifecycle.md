# データのライフサイクル、測定、偏り、倫理

## 学習目標

1. 問いが、測定、収集、保存、利用、削除されるデータになる流れを説明できる。（D4.O1）
2. 標本抽出、測定、欠測、表現の偏りを識別できる。（D4.O2）
3. 目的、同意、プライバシー、データ最小化を考慮した収集を計画できる。（D4.O3）

## 前提知識

- A2で学んだ、情報源、根拠、確かさ、限界の評価。
- A4で学んだ、問題、対象者、目的、要件の定義。

## 導入: 集計の前に、データが生まれる過程を見る

架空の星見高校図書室で「放課後の座席は足りているか」を調べるとします。いきなり人数を集めても、「放課後」「利用者」「足りる」の定義が違えば、同じ問いに答えられません。

この章の人数、時刻、回答はすべて合成データです。実在する生徒の行動、氏名、端末識別子は含みません。

## 問いを測定可能にする

大きな問いを、記録できる定義へ変えます。

| 段階 | 合成計画 |
| --- | --- |
| 問い | 放課後に図書室の座席が不足する時間帯はあるか |
| 対象 | 調査対象日に図書室を利用する人数と空席数 |
| 時間 | 15:30、16:00、16:30の3時点 |
| 測定 | 室内人数と利用可能な座席数を数える |
| 判断の目安 | 空席が2席未満の時点を「混雑」とする |

「満足度」のように直接数えにくい概念は、質問文や回答尺度へ置き換えます。その置き換えが元の概念を完全に表すとは限らないため、定義と限界を残します。

## データのライフサイクル

データは集めた瞬間だけでなく、前後の段階で責任が生じます。

1. **目的を決める**: 何を判断するためか。
2. **測定を設計する**: 誰を、何を、いつ、どの単位で測るか。
3. **収集する**: 必要な範囲だけを、説明した方法で記録する。
4. **保存する**: 保存場所、期間、アクセスできる人を決める。
5. **整形・分析する**: 変換履歴を残し、問いに合う方法を使う。
6. **利用・共有する**: 当初の目的、同意、公開範囲と合うか確かめる。
7. **保持・削除を判断する**: 必要がなくなったデータを漫然と残さない。

後から別目的に使える可能性があっても、それだけで無制限に収集・保存してよい理由にはなりません。

## 四つの偏りを見分ける

| 偏り | どこで生じるか | 合成事例 |
| --- | --- | --- |
| 標本抽出の偏り | 誰・いつを調べるか | 月曜だけ測って全曜日へ一般化する |
| 測定の偏り | 測り方や質問 | 出入口の一部だけで人数を数える |
| 欠測の偏り | 記録されなかった値 | 混雑時だけ記録を忘れる |
| 表現の偏り | 分類・集計・表示 | 15:30と16:30をまとめ、16:00の混雑を隠す |

偏りがあるからデータが無価値とは限りません。どの方向へ判断をずらし得るかを説明し、追加調査や表現の修正を考えます。

## 必要最小限のデータを計画する

座席数を調べる目的なら、個人名や端末識別子は不要です。次の比較を行います。

| 候補項目 | 目的への必要性 | 判断 |
| --- | --- | --- |
| 観測日時 | 時間帯を比較するため必要 | 収集する |
| 室内人数 | 混雑を判断するため必要 | 収集する |
| 利用可能席数 | 不足を判断するため必要 | 収集する |
| 生徒氏名 | 座席不足の判断には不要 | 収集しない |
| 閲覧した本 | 座席不足の判断には不要 | 収集しない |

これが**データ最小化**の考え方です。収集前に、目的、説明・同意の要否、保存期間、アクセス範囲、削除方法を確認します。法的な適用や学校規程への適合は事実関係で変わるため、教材だけで結論を出さず担当者が確認します。

## 収集計画を批判的に読む

悪い計画: 「便利そうなので、1年間、図書室に入った全員の氏名、時刻、端末ID、閲覧資料を保存する。」

改善すると次のようになります。

- 判断したい問いを座席不足へ限定する。
- 氏名、端末ID、閲覧資料を収集しない。
- 複数の曜日と時間帯を同じ規則で観測する。
- 混雑時にも記録できる手順を決める。
- 集計後に元の観測記録をいつ削除するか決める。
- 利用者へ調査目的と問い合わせ先を示す必要性を確認する。

## よくある間違い

- 集められる項目はすべて集めた方がよいと考える。
- 人数が多ければ偏りがないと考える。測り方や対象時間の偏りは残ります。
- 欠測値を単純に0とみなす。0と「記録なし」は意味が違います。
- 匿名ならどんな再利用もできると考える。目的、再識別リスク、説明範囲を確認します。
- 相関や傾向が見えた後に、当初の問いを都合よく書き換える。

## 自己確認

1. 問いから削除までの七段階を、座席調査へ対応付けられますか。
2. 四つの偏りを別々の例で説明できますか。
3. 収集しない項目と、その理由を明記できますか。

## 自己確認の診断と修正

- 収集から始めた場合は、先に判断目的と測定定義を書きます。
- 「偏っている」で止まった場合は、どの段階で、どちらの方向へずれるかを説明します。
- プライバシー対策が名前の削除だけの場合は、時刻、識別子、組合せによる特定可能性も検討します。

## まとめ

- データは、目的、測定、収集、保存、分析、利用、削除のライフサイクルで管理します。
- 標本抽出、測定、欠測、表現の偏りは、判断を異なる仕方でずらします。
- 必要最小限の項目、期間、権限を収集前に決め、目的と限界を記録します。

## 練習

- **問いからデータの流れを設計する**
- **四つの偏りを識別する**
- **必要最小限の収集計画へ直す**
- **合成調査計画を批判し改善する**
