# インターネット、Web、情報サービス

## 学習目標

1. インターネット、Web、情報サービスを区別できる。（D2.O1）
2. 名前解決、要求と応答、データ処理、サービス提供の単純化した流れをたどれる。（D2.O2）
3. 情報システムの構成と運用が、サービス利用をどう支え、制約するか評価できる。（D2.O3）

## 前提知識

- D1で学んだ、端末、伝送路、中継機器、プロトコル、パケット。

## 導入: Webページだけがインターネットではない

ブラウザーで図書検索画面を開くとき、利用者が見ているのはWeb上の情報です。しかし、その背後では名前の問い合わせ、ネットワーク上の転送、サーバーでの検索、応答の表示が協力しています。

## 三つの言葉を区別する

| 用語 | この教材での意味 | 例 |
| --- | --- | --- |
| インターネット | 複数のネットワークを共通の規則で相互接続した基盤 | パケットを端末から遠隔のサーバーへ運ぶ |
| Web | URI（資源を識別する文字列）で情報を指し示し、HTTPなどで取得・参照できる情報空間 | ブラウザーで検索画面を取得する |
| 情報サービス | 利用者の目的を満たすため、データ、処理、人、運用を組み合わせた働き | 蔵書を検索し、貸出可否を返す |

Webはインターネットを利用する仕組みの一つです。電子メールなど、インターネットを使うがWebそのものではないサービスもあります。また、情報サービスは画面だけではありません。サーバー、データベース、運用手順、問い合わせ対応などを含みます。

## 名前からサービスへ到達する流れ

架空のアドレス `library.example.test` で蔵書 `B204` を検索する流れを単純化します。このアドレスは教材内の例示にだけ使い、実在するサービスへの接続には使いません。

1. **名前解決**: 端末は、名前に対応する接続先を問い合わせる。
2. **接続**: 端末とWebサーバーが通信経路を使える状態にする。
3. **要求**: ブラウザーが「蔵書ID `B204` の情報がほしい」と送る。
4. **処理**: サーバーが要求を検査し、データベースへ問い合わせる。
5. **応答**: サーバーが結果と状態を返す。
6. **提示**: ブラウザーが、見出しや表として利用者へ示す。

これは学習用の単純化です。実際にはキャッシュ、暗号化、複数サーバー、負荷分散などが加わる場合があります。

## どこで失敗したかを切り分ける

「検索できない」という同じ見た目でも、原因は異なります。

| 観察 | 候補となる箇所 | 次に確かめること |
| --- | --- | --- |
| 名前だけ解決できない | 名前解決 | 別の既知の名前は解決できるか |
| 画面は開くが検索結果が返らない | アプリケーションまたはデータベース | 状態表示、要求形式、データの有無 |
| 一部の端末だけ使えない | 端末またはその接続区間 | 同じ経路の別端末との差 |
| 結果が古い | データ更新またはキャッシュ | 更新時刻と参照したデータ |

一度にすべてを変えると、どの変更が効いたか分かりません。流れを段階へ分け、一つずつ観察します。

## 情報システムをサービスとして評価する

サービスが動作するだけでは、目的を十分に満たすとは限りません。次の観点を、利用者の必要と結び付けます。

- **有効性**: 目的に合う情報を誤りなく得られるか。
- **利用可能性**: 必要な時間と場所で使えるか。
- **性能**: 許容できる時間で応答するか。
- **アクセシビリティ**: キーボード、読み上げ、文字拡大などで利用できるか。
- **運用性**: 更新、障害対応、問い合わせの担当が決まっているか。
- **安全性**: 権限のない閲覧や変更を抑えられるか。

たとえば応答を速くするために古い結果を長く保存すると、速度は上がっても最新性が下がる場合があります。評価では一つの指標だけで結論を出さず、目的とトレードオフを明示します。

## よくある間違い

- インターネットとWebを同じ意味で使う。
- 情報サービスを画面だけと考え、データや運用を含めない。
- サーバーが一台の物理機器だと決めつける。役割が複数機器へ分かれる場合があります。
- 「つながらない」をすぐネットワーク障害と断定する。要求、処理、データにも原因があり得ます。
- 応答速度だけでサービス品質を判断する。情報の誤り、利用可能性、安全性なども確認します。

## 自己確認

1. 電子メールがインターネットを使ってもWebとは限らない理由を説明できますか。
2. 名前解決から画面表示までを六段階でたどれますか。
3. 速さを改善した結果、最新性が下がる例を評価できますか。

## 自己確認の診断と修正

- 用語が混ざる場合は、「基盤」「情報空間」「目的を満たす働き」に分けます。
- 障害原因を一つに決めた場合は、各段階で観察できる証拠を書きます。
- 評価が「便利」で止まった場合は、誰のどの必要を、どの指標で満たすかを具体化します。

## まとめ

- インターネットは接続基盤、Webはその上の情報空間、情報サービスは利用目的を満たす仕組みです。
- 名前解決、要求、処理、応答、提示へ分けると、仕組みと障害箇所を説明できます。
- 情報システムは、構成要素と運用を含め、複数の品質観点で評価します。

- **情報サービスを要件から評価する**
