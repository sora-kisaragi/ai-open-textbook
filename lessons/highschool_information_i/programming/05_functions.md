# 関数と分解

## 学習目標

1. 関数を、入力を受け取り、決まった処理を行い、結果または作用を生む、名前の付いた再利用可能な操作として説明できる。（C6.O1）
2. 短いPythonの関数を定義し、呼び出せる。（C6.O2）
3. 一つの問題を、責務が明確な関数へ分解できる。（C6.O3）

## 前提知識

- 変数と代入、単純な `if` / `else`、繰り返しを使えること。
- リストと、表の一行を表す単純なリストを添字で読めること。

## 印刷依頼の処理を分けて考える

架空の星見高校メディアラボでは、依頼番号、1部のページ数、部数を記録します。`ページ数 × 部数` で求めた紙の数を**枚数**と呼びます。両面印刷は扱わず、1ページを紙1枚に印刷する単純なモデルです。記録はすべて教材用の合成データです。

「依頼を処理する」という仕事を、次の責務へ分けます。

1. ページ数と部数が1以上か確かめる。
2. 必要枚数を計算する。
3. 複数の依頼を順に処理して表示する。

| 操作 | 入力 | 戻り値または作用 |
| --- | --- | --- |
| 依頼を検査する | ページ数、部数 | 有効なら `True`、無効なら `False` |
| 枚数を計算する | ページ数、部数 | 必要枚数 |
| 一覧を処理する | 依頼を表すリスト行の集まり | 各依頼の結果を表示する |

**引数**は関数へ渡す入力、**戻り値**は関数が呼び出し元へ返す結果です。関数の外にある値を隠れて使わず、必要な値を引数で渡します。

## 短い関数を定義して呼び出す

```python
def sheet_count(pages, copies):
    sheets = pages * copies
    return sheets


result = sheet_count(4, 3)
print(result)
```

出力は `12` です。呼び出しを一段ずつたどると、次のようになります。

| 段階 | 起きること | 値 |
| ---: | --- | ---: |
| 1 | `sheet_count(4, 3)` を呼ぶ | `pages` は4、`copies` は3 |
| 2 | `pages * copies` を計算する | 12 |
| 3 | `return sheets` で返す | 12 |
| 4 | 戻り値を `result` に代入する | 12 |
| 5 | `result` を表示する | 12 |

`print` は画面へ表示し、`return` は呼び出し元へ値を返します。役割を区別します。

## C5までの構文で複数の依頼を処理する

複数の条件を一行へ詰めず、習った単純な分岐を組み合わせます。依頼は `[依頼番号, ページ数, 部数]` というリスト行で表します。

```python
def is_valid_job(pages, copies):
    if pages < 1:
        return False
    else:
        if copies < 1:
            return False
        else:
            return True


def sheet_count(pages, copies):
    return pages * copies


jobs = [
    ["M201", 2, 15],
    ["M202", 0, 8],
    ["M203", 4, 3],
]

for job in jobs:
    job_id = job[0]
    pages = job[1]
    copies = job[2]
    if is_valid_job(pages, copies):
        sheets = sheet_count(pages, copies)
        print(job_id, sheets, "枚")
    else:
        print(job_id, "入力エラー")
```

実行結果は次のとおりです。

```text
M201 30 枚
M202 入力エラー
M203 12 枚
```

この例で使うのは、値を受け取る関数、単純な `if` / `else`、リスト行、添字、繰り返しだけです。

## 呼び出しを明示的にたどる

`M203` の行 `["M203", 4, 3]` を処理するときの流れです。

1. 添字0、1、2から `M203`、4、3を取り出す。
2. `is_valid_job(4, 3)` を呼ぶ。
3. 4は1未満ではなく、3も1未満ではないので `True` が返る。
4. `sheet_count(4, 3)` を呼び、12が返る。
5. `M203 12 枚` を表示する。

途中で値が分からなくなったら、「呼び出し前の引数」「関数内の計算」「戻り値」「呼び出し後の利用」の四つに分けます。

## リファクタリングの前と後

**変更前**は同じ計算が繰り返されています。

```python
pages = 2
copies = 3
first_sheets = pages * copies

pages = 5
copies = 2
second_sheets = pages * copies

print(first_sheets)
print(second_sheets)
```

**変更後**は、同じ責務に名前を付けて一か所へまとめます。

```python
def sheet_count(pages, copies):
    return pages * copies


first_sheets = sheet_count(2, 3)
second_sheets = sheet_count(5, 2)

print(first_sheets)
print(second_sheets)
```

この例では、結果を変えずに計算を関数へまとめています。外から見える動作を保ちながらコードの構造を整理することを**リファクタリング**といいます。この例の計算規則を変更するときは、`sheet_count` を中心に確認できます。

## 期待値と実際値で小さく確かめる

```python
def sheet_count(pages, copies):
    return pages * copies


expected = 12
actual = sheet_count(4, 3)
print("期待値:", expected)
print("実際値:", actual)
print("一致:", expected == actual)
```

一つの確認が失敗したときは、期待値、実際値、呼び出した関数を記録します。エラーが出なかったことだけで正しいとは判断しません。

## よくある間違い

- 関数名だけを書き、丸括弧を付けずに呼び出したつもりになる。
- `print` した値が、自動的に戻り値になると考える。
- `return` で返した値を受け取らず、その後の処理に使えない。
- 関数の外にある変数へ密かに依存させる。必要な値は引数で渡します。
- 同じ計算を関数の内外へ重複して書く。
- リファクタリング前後で入力を変え、結果が同じか確認できなくする。

## 自己確認

- `sheet_count` の入力と戻り値を言葉で説明できますか。
- `sheet_count(5, 2)` の呼び出しを段階ごとにたどれますか。
- 検査と計算を別の関数にする利点を説明できますか。
- 変更前後で結果が同じか、期待値と実際値で確認できますか。

## まとめ

- 関数は、名前、入力、処理、戻り値または作用を持つ再利用可能な操作です。
- 引数、関数内の処理、戻り値、呼び出し後の利用を順にたどります。
- この例では、責務ごとに分けることで重複を減らし、関数ごとに確かめられます。
- リファクタリングでは結果を保ったまま構造を改善し、前後を同じ入力で確認します。

## 練習

- **印刷枚数を返す関数を作る**
- **重複した処理を責務ごとの関数へ分ける**
- **印刷依頼処理を複数の関数で構成する**
