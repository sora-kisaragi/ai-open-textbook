# モデル化とシミュレーション

## 学習目標

1. 簡単なモデルの変数、仮定、関係、範囲を特定できる。（C8.O1）
2. 同じモデルを手作業とPythonでシミュレーションできる。（C8.O2）
3. パラメータの変更が結果へ与える影響と、モデルの限界を評価できる。（C8.O3）

## 前提知識

- 問題を分解し、モデルが意図的に除外するものを説明できること。
- 単純な条件分岐、繰り返し、リスト行、添字、関数を使えること。

## 目的に合うモデルを作る

星見高校メディアラボでは、昼休みの印刷待ちが長くなる条件を調べます。実際のプリンターや生徒の記録は使わず、教材用の合成データで待ち行列を再現します。

**モデル**は、目的に関係する現実の一部を選んだ表現です。**シミュレーション**は、そのモデルの規則に従って状態の変化をたどることです。結果は現実の予言ではありません。

### 今回のモデル

- **変数**: 到着時刻、必要枚数、開始時刻、終了時刻、待ち時間。
- **パラメータ**: 1分に印刷できる枚数 `speed`。
- **関係**: プリンターが空いていれば到着時刻に開始し、使用中なら前の依頼の終了後に開始する。
- **仮定**: 一台で受付順に処理する。途中停止と切替時間はない。
- **範囲**: 12時00分を0分とした固定4件だけを扱う。
- **除外**: 紙詰まり、用紙補充、通信遅延、依頼の取消し、両面印刷、個人情報。

必要枚数はすでに「枚」で与えます。`ページ数 × 部数` を使う場合も結果は枚数であり、両面印刷はこのモデルの外です。

## 手作業で規則をたどる

速度を1分4枚とします。最初の依頼 `M401` は0分に到着し、必要枚数は8枚です。

1. プリンターは空いているので、開始時刻は0分。
2. `8 ÷ 4 = 2` なので処理時間は2分。
3. 終了時刻は2分。
4. 待ち時間は `開始0 - 到着0 = 0` 分。

次の `M402` は1分に到着しますが、前の依頼が2分に終わるまで待つので、開始時刻は2分、待ち時間は1分です。

## リスト行と単純な制御で実装する

依頼は `[ID, 到着時刻, 必要枚数]` というリスト行で表します。`ceil_divide` は、割り切れない処理時間を1分単位で切り上げます。

```python
def ceil_divide(value, divisor):
    return (value + divisor - 1) // divisor


def simulate_fifo(jobs, speed):
    if speed < 1:
        print("速度は1以上にしてください")
        return -1

    printer_free_at = 0
    total_wait = 0
    count = 0

    for job in jobs:
        job_id = job[0]
        arrival = job[1]
        sheets = job[2]

        if arrival > printer_free_at:
            start = arrival
        else:
            start = printer_free_at

        duration = ceil_divide(sheets, speed)
        finish = start + duration
        wait = start - arrival
        print(job_id, start, finish, wait)

        printer_free_at = finish
        total_wait = total_wait + wait
        count = count + 1

    if count == 0:
        return 0
    else:
        return total_wait / count


jobs = [
    ["M401", 0, 8],
    ["M402", 1, 12],
    ["M403", 4, 4],
    ["M404", 5, 16],
]

average = simulate_fifo(jobs, 4)
print("平均待ち時間:", average)
```

表示の各行は `ID 開始 終了 待ち時間` です。

```text
M401 0 2 0
M402 2 5 1
M403 5 6 1
M404 6 10 1
平均待ち時間: 0.75
```

速度が0以下なら計算をせず、`-1` を返します。無効な速度と空の依頼一覧を同じ結果にしないためです。空の依頼一覧の平均は、この教材では0と定義します。

## 手作業とPythonを照合する

同じ入力と同じ規則で、手作業とPythonの値を比べます。

| ID | 手作業の開始 | Pythonの開始 | 手作業の待ち | Pythonの待ち |
| --- | ---: | ---: | ---: | ---: |
| `M401` | 0 | 0 | 0 | 0 |
| `M402` | 2 | 2 | 1 | 1 |
| `M403` | 5 | 5 | 1 | 1 |
| `M404` | 6 | 6 | 1 | 1 |

一致しない場合は、到着時刻と前の終了時刻のどちらを開始時刻にしたか、切り上げ、待ち時間の引き算を順に確認します。

## 速度だけを変えて感度を見る

このレッスンでは、他の条件を固定して速度だけを変え、結果がどのように変わるかを調べます。この調べ方を**感度分析**と呼びます。同じ `jobs` を速度2、4、8枚/分で実行して得た待ち時間を、独立したコードで集計します。

```python
sensitivity_rows = [
    [2, [0, 3, 6, 7]],
    [4, [0, 1, 1, 1]],
    [8, [0, 0, 0, 0]],
]

for row in sensitivity_rows:
    speed = row[0]
    waits = row[1]
    total_wait = 0
    count = 0
    for wait in waits:
        total_wait = total_wait + wait
        count = count + 1
    average = total_wait / count
    print(speed, average)
```

期待する待ち時間と平均は次のとおりです。

| 速度（枚/分） | 各依頼の待ち時間 | 平均待ち時間（分） |
| ---: | --- | ---: |
| 2 | `[0, 3, 6, 7]` | 4.0 |
| 4 | `[0, 1, 1, 1]` | 0.75 |
| 8 | `[0, 0, 0, 0]` | 0.0 |

速度2のときは、待ち時間の合計が16分、依頼が4件なので、平均は `16 ÷ 4 = 4.0` 分です。Pythonの表示もこの手計算と照合します。

この固定データでは速度が上がるほど平均待ち時間は小さくなります。ただし、どんな日でも同じ割合で減るとは言えません。到着間隔や枚数、除外した故障などが変われば結果も変わります。

## モデルを評価する

このモデルは、固定された依頼に対する速度と待ち時間の関係を説明できます。一方、実際の機器購入や運用変更を決める証拠としては不十分です。

- 合成データ4件だけで、日ごとの変動を表していない。
- 処理時間を整数分へ切り上げている。
- 故障、切替、用紙補充、両面印刷を除外している。
- 受付順以外の規則を中核では比較していない。

改善する場合は、一度に一つの仮定だけを変え、変更前後で同じ入力を再テストします。

## よくある間違い

- モデルの数値を、現実で必ず起こる結果として扱う。
- 到着時刻だけで開始時刻を決め、前の依頼の終了を見ない。
- 複数のパラメータを同時に変え、原因を説明できなくする。
- 速度2の待ち時間 `[0, 3, 6, 7]` を合計せず、平均を推測する。
- 無効な速度と空の依頼一覧を同じ意味として扱う。
