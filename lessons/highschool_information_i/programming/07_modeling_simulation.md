# モデル化とシミュレーション

## 学習目標

1. 簡単なモデルの変数、仮定、関係、範囲を特定できる。（C8.O1）
2. 同じモデルを手作業とPythonでシミュレーションできる。（C8.O2）
3. 同じ入力に対する二つの決定論的な仮定セットを比較し、結果と限界を評価できる。（C8.O3）

## 前提知識

- 問題を分解し、モデルが意図的に除外するものを説明できること。
- 単純な条件分岐、繰り返し、リスト行、添字、関数を使えること。

## 準備チェックと復習

「到着時刻」「処理時間」「天候」を、待ち時間モデルへ含める変数と今回は除外する要因へ分けます。目的と範囲を説明できなければC6の分解へ戻り、入力と出力を絞ってから再挑戦します。迷ったら一件だけを使い、目的に必要な入力と結果を確認してから複数件へ戻ります。

## 考え方のモデル

モデルは「目的→変数・パラメータ→関係→仮定→範囲・除外」で表します。シミュレーションは同じ規則で状態を更新する実験です。結果、仮定、現実を分け、仮定を一つ変えた比較で感度と限界を調べます。

## 例題: 二つの仮定を比べる

到着0分、処理3分の二件を順に扱います。準備時間0分なら終了は3分と6分、各件に1分なら4分と8分です。同じ入力で準備時間だけを変えたため、差をその仮定へ結び付けられます。

## 一緒に試す

到着0、2、5分、処理3分の窓口を表で追います。開始、終了、待ち時間を一行ずつ更新し、処理時間を4分へ変えた二案の差と、モデルに含めない要因を一つ示します。

## 目的に合うモデルを作る

星見高校メディアラボでは、昼休みの印刷待ちが長くなる条件を調べます。実際のプリンターや生徒の記録は使わず、教材用の合成データで待ち行列を再現します。

**モデル**は、目的に関係する現実の一部を選んだ表現です。**シミュレーション**は、そのモデルの規則に従って状態の変化をたどることです。結果は現実の予言ではありません。

### 今回のモデル

- **変数**: 到着時刻、必要枚数、開始時刻、終了時刻、待ち時間。
- **パラメータ**: 1分に印刷できる枚数 `speed` と、1件ごとの準備時間 `setup`。
- **関係**: プリンターが空いていれば到着時刻に開始し、使用中なら前の依頼の終了後に開始する。
- **仮定**: 一台で受付順に処理する。途中停止はない。準備時間は比較するモデルごとに固定する。
- **範囲**: 12時00分を0分とした固定4件だけを扱う。
- **除外**: 紙詰まり、用紙補充、通信遅延、依頼の取消し、両面印刷、個人情報。

必要枚数はすでに「枚」で与えます。`ページ数 × 部数` を使う場合も結果は枚数であり、両面印刷はこのモデルの外です。

## 手作業で規則をたどる

速度を1分4枚とします。最初の依頼 `M401` は0分に到着し、必要枚数は8枚です。

1. プリンターは空いているので、開始時刻は0分。
2. `8 ÷ 4 = 2` なので処理時間は2分。
3. 終了時刻は2分。
4. 待ち時間は `開始0 - 到着0 = 0` 分。

次の `M402` は1分に到着しますが、前の依頼が2分に終わるまで待つので、開始時刻は2分、待ち時間は1分です。

## リスト行と単純な制御で実装する

依頼は `[ID, 到着時刻, 必要枚数]` というリスト行で表します。`ceil_divide` は、割り切れない処理時間を1分単位で切り上げます。

```python
def ceil_divide(value, divisor):
    return (value + divisor - 1) // divisor


def simulate_fifo(jobs, speed, setup):
    if speed < 1:
        print("速度は1以上にしてください")
        return -1

    printer_free_at = 0
    total_wait = 0
    count = 0

    for job in jobs:
        job_id = job[0]
        arrival = job[1]
        sheets = job[2]

        if arrival > printer_free_at:
            start = arrival
        else:
            start = printer_free_at

        duration = ceil_divide(sheets, speed) + setup
        finish = start + duration
        wait = start - arrival
        print(job_id, start, finish, wait)

        printer_free_at = finish
        total_wait = total_wait + wait
        count = count + 1

    if count == 0:
        return 0
    else:
        return total_wait / count


jobs = [
    ["M401", 0, 8],
    ["M402", 1, 12],
    ["M403", 4, 4],
    ["M404", 5, 16],
]

average = simulate_fifo(jobs, 4, 0)
print("平均待ち時間:", average)
```

表示の各行は `ID 開始 終了 待ち時間` です。

```text
M401 0 2 0
M402 2 5 1
M403 5 6 1
M404 6 10 1
平均待ち時間: 0.75
```

速度が0以下なら計算をせず、`-1` を返します。無効な速度と空の依頼一覧を同じ結果にしないためです。空の依頼一覧の平均は、この教材では0と定義します。

## 手作業とPythonを照合する

同じ入力と同じ規則で、手作業とPythonの値を比べます。

| ID | 手作業の開始 | Pythonの開始 | 手作業の待ち | Pythonの待ち |
| --- | ---: | ---: | ---: | ---: |
| `M401` | 0 | 0 | 0 | 0 |
| `M402` | 2 | 2 | 1 | 1 |
| `M403` | 5 | 5 | 1 | 1 |
| `M404` | 6 | 6 | 1 | 1 |

一致しない場合は、到着時刻と前の終了時刻のどちらを開始時刻にしたか、切り上げ、待ち時間の引き算を順に確認します。

## 二つの決定論的モデルを比較する

同じ合成データと速度4枚/分を使い、準備時間の仮定だけが異なる二つのモデルを比較します。どちらも乱数を使わず、同じ入力なら必ず同じ結果になる**決定論的モデル**です。

- **モデルA**: 1件ごとの準備時間は0分。
- **モデルB**: 1件ごとに固定1分の準備時間を加える。

モデルBでは、処理時間を `切り上げた印刷時間 + 1分` とします。最初の依頼にも準備時間を含めます。

| モデル | 準備時間 | 各依頼の待ち時間 | 平均待ち時間 |
| --- | ---: | --- | ---: |
| A | 0分 | `[0, 1, 1, 1]` | 0.75分 |
| B | 1分 | `[0, 2, 3, 4]` | 2.25分 |

モデルBの開始・終了・待ち時間を手作業でたどると、次のようになります。

| ID | 開始 | 終了 | 待ち |
| --- | ---: | ---: | ---: |
| `M401` | 0 | 3 | 0 |
| `M402` | 3 | 7 | 2 |
| `M403` | 7 | 9 | 3 |
| `M404` | 9 | 14 | 4 |

Pythonでは、同じ `jobs` に対して `simulate_fifo(jobs, 4, 0)` と `simulate_fifo(jobs, 4, 1)` を実行し、各行と平均を手計算へ照合します。変えたのは準備時間の仮定だけです。

この比較から、「この固定入力と規則では、準備時間を含むモデルBの平均待ち時間が大きい」と言えます。一方、現実の準備時間が必ず1分であることや、別の日にも差が1.5分になることは言えません。

確率を使うモデルは、到着間隔などのばらつきを扱う発展課題にできます。ただし、このレッスンの中核では、二つの決定論的な仮定セットを正確に比較できれば十分です。

## モデルを評価する

このモデルは、固定された依頼に対する速度と待ち時間の関係を説明できます。一方、実際の機器購入や運用変更を決める証拠としては不十分です。

- 合成データ4件だけで、日ごとの変動を表していない。
- 処理時間を整数分へ切り上げている。
- 故障、切替、用紙補充、両面印刷を除外している。
- 受付順以外の規則を中核では比較していない。

改善する場合は、一度に一つの仮定だけを変え、変更前後で同じ入力を再テストします。

## よくある間違い

- モデルの数値を、現実で必ず起こる結果として扱う。
- 到着時刻だけで開始時刻を決め、前の依頼の終了を見ない。
- 入力と複数の仮定を同時に変え、結果の差を何が生んだか説明できなくする。
- モデルAとBのどちらに準備時間を含めたか記録しない。
- 無効な速度と空の依頼一覧を同じ意味として扱う。

## 自己確認

- モデルの変数、パラメータ、仮定をそれぞれ言えますか。
- `M402` の開始、終了、待ち時間を手作業で説明できますか。
- モデルBの平均待ち時間2.25を手計算とPythonで照合できますか。
- 二つのモデルで固定したものと変えた仮定を区別できますか。
- このモデルから言えることと言えないことを書けますか。

## まとめ

- モデルは目的に関係する部分を選び、変数、関係、仮定、範囲、除外を明示します。
- リスト行、単純な分岐、繰り返し、カウンタ、関数で待ち行列を再現できます。
- 手作業とPythonを同じ入力で照合し、規則どおりか確かめます。
- 二つの決定論的モデルでは仮定を一つだけ変え、結果と限界を合わせて評価します。

## 別の場面への転移

窓口の待ち時間を、到着間隔と処理時間だけを使う簡単なモデルで考えます。Pythonを使わず表を一行ずつ更新し、二つの仮定で結果を比較して、モデルに含めなかった要因と適用できる範囲を説明してください。

これは2時限の授業外で行う25分課題です。20分で目的、変数、仮定、二案の状態表、差、限界をA4判1枚にまとめます。続く5分で固定した入力、変えた仮定、手計算、適用範囲を照合し、一箇所を修正します。

## 到達チェックと再挑戦

同じ入力で一仮定だけを変え、結果差と限界を分けて示せれば到達です。ずれた場合は一件だけの状態更新へ戻り、開始と終了を照合してから複数件へ再挑戦します。
