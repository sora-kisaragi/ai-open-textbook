# アルゴリズムと効率

## 前提知識

- 条件、関数の入力と戻り値を説明できること。

## 準備チェックと復習

番号カード `[6, 2, 5, 3]` の先頭から5を探し、比較した順を記録します。途中で止める条件と「ない」と判断する時点を説明できなければ、C4の停止条件とC5の添字を確認して再挑戦します。迷ったら三枚だけに減らし、比較したカードへ印を付けてから元の課題へ戻ります。

## 考え方のモデル

アルゴリズムは「入力、順序付きの有限手順、終了条件、出力」の契約です。自然言語または表で再現できる形にし、途中状態と比較回数を追い、通常・境界・空で正しさを確かめてからPythonへ対応付けます。

## 例題: 探索を手で追う

`[6, 2, 5, 3]` から5を探します。6、2、5の順に3回比較し、添字2で終了します。7を探す場合は4回比較して末尾へ達した後に「なし」とします。

## 一緒に試す

カード `[4, 1, 3]` を選択ソートします。未整列部分の最小値、交換後の並び、整列済み境界を各回記録し、同じ値を含む場合にも手順が終了するかを確かめます。

## アルゴリズムを再現可能な手順にする

このレッスンでは、**アルゴリズム**を「与えられた入力から目的の出力を得るために、実行順序と終了条件を明確にした有限の手順」として扱います。

同じ入力なら、別の人が手順をたどって同じ結果を得られる必要があります。「適切な項目を探す」「よい順に並べる」だけでは、何をどの順で調べるかが決まりません。

## 線形探索: 先頭から一つずつ探す

架空の備品番号が、次の順で記録されています。

```text
[K14, M03, F22, H08]
```

`F22` を探す**線形探索**は、先頭から一つずつ目標と比べます。

```text
位置を先頭にする
未確認の項目がある間、次を繰り返す
    現在の項目と目標を比べる
    等しければ、その位置を答えて終了する
    等しくなければ、次の位置へ進む
最後まで見つからなければ「なし」と答える
```

| 比較回数 | 現在の値 | `F22` と等しいか | 次の動作 |
| ---: | --- | --- | --- |
| 1 | `K14` | いいえ | 次へ進む |
| 2 | `M03` | いいえ | 次へ進む |
| 3 | `F22` | はい | 3番目と答えて終了 |

見つかった後も調べ続ける必要はありません。目標がない場合だけ、4件すべてを比べます。0件なら比較せず「なし」です。

## 線形探索の正しさと仕事量

線形探索が正しいと言えるためには、次を確かめます。

1. 先頭から順に、飛ばさず調べる。
2. 一致した位置を返す。
3. 一致しないまま末尾へ達したときだけ「なし」とする。
4. 有限個の項目を一度ずつ進むので必ず終了する。

最悪の場合の比較回数は、4件なら4回、8件なら8回です。入力が2倍になると、最悪の比較回数もおおむね2倍になります。

## 選択ソート: 未整列部分の最小値を選ぶ

次に、処理時間 `[6, 2, 5, 3]` を小さい順へ並べます。ここでは、**選択ソート**を使います。

```text
左端を整列済み部分の境界とする
未整列部分が2個以上ある間、次を繰り返す
    未整列部分をすべて調べ、最小値の位置を記録する
    最小値を未整列部分の左端と入れ替える
    整列済み部分を一つ右へ広げる
```

| 回 | 開始時 | 未整列部分の最小値 | 入れ替え後 |
| ---: | --- | ---: | --- |
| 1 | `[6, 2, 5, 3]` | 2 | `[2, 6, 5, 3]` |
| 2 | `[2, 6, 5, 3]` | 3 | `[2, 3, 5, 6]` |
| 3 | `[2, 3, 5, 6]` | 5 | `[2, 3, 5, 6]` |

各回の終了時には、整列済み部分が小さい順になり、そこには残りの値より大きい値が入りません。未整列部分をすべて調べて最小値を選ぶため、この性質を一つずつ広げられます。最後に未整列の値が一つになれば、全体が整列済みです。

## 選択ソートの仕事量

4個では、最小値を探す比較が `3 + 2 + 1 = 6` 回です。8個では `7 + 6 + 5 + 4 + 3 + 2 + 1 = 28` 回です。

| 入力数 | 比較回数 |
| ---: | ---: |
| 4 | 6 |
| 8 | 28 |

この例では入力が4個から8個へ2倍になると、比較回数は2倍より大きく増えます。正式な計算量表記を暗記することより、「どの繰り返しが何回増えるか」をトレースから説明することを重視します。

## 二つの手順を使い分ける

線形探索は、並びを変えずに目標を一つ探します。選択ソートは、全体を指定した順へ並べ替えます。目的も出力も異なります。

- 受付順を保った一覧から一つのIDを探す: 線形探索を使える。
- 数値を小さい順へ並べた新しい一覧が必要: 選択ソートを使える。
- 一つ探すだけなのに毎回全体を並べ替える: 余分な仕事が増える場合がある。

アルゴリズムを選ぶときは、入力、必要な出力、元の順序を変えてよいかを先に確認します。

## Pythonで確かめる

概念を手作業で確認した後、線形探索をPythonで表すと次のようになります。

```python
def linear_search(values, target):
    position = 0
    for value in values:
        if value == target:
            return position
        position = position + 1
    return -1


codes = ["K14", "M03", "F22", "H08"]
print(linear_search(codes, "F22"))
print(linear_search(codes, "Z99"))
```

表示は0から数えた位置なので、`2` と `-1` です。自然言語の「3番目」とPythonの添字 `2` を混同しないようにします。

## テストと改善

期待値を実行前に決めます。

| 種類 | 入力 | 期待値 |
| --- | --- | --- |
| 先頭 | `[K14, M03, F22, H08]` から `K14` | 1回で見つかる |
| 途中 | 同じ一覧から `F22` | 3回で見つかる |
| なし | 同じ一覧から `Z99` | 4回比較して「なし」 |
| 空 | `[]` から `K14` | 0回で「なし」 |

誤った手順が最初の不一致で「なし」と判断するなら、先頭のテストだけでは見つけられません。途中、なし、空を組み合わせ、期待値と実際値を比べます。

## よくある間違い

- 線形探索で最初の不一致を見て「なし」とする。末尾まで調べてから判断します。
- 見つかった後も全件を調べる。必要な出力が最初の位置なら、その場で終了できます。
- 選択ソートで未整列部分の一部だけを見て最小値とする。境界から末尾まで比較します。
- 値を入れ替えた後、整列済み部分の境界を更新しない。
- コードの行数だけで効率を判断する。入力数と、比較や更新の回数を結び付けます。

## まとめ

- 線形探索は、先頭から一つずつ比べ、見つかれば終了し、末尾までなければ「なし」とします。
- 選択ソートは、未整列部分の最小値を選び、整列済み部分を一つずつ広げます。
- トレースでは途中状態、比較回数、終了条件を記録し、正しさを観察します。
- 入力が増えたとき、線形探索と選択ソートでは仕事量の増え方が異なります。

## 別の場面への転移

これは2時限の授業外で行う25分課題です。20分で二手順、途中状態、三テスト、比較回数をA4判1枚にまとめます。続く5分で有限性、終了、期待結果、仕事量の根拠を照合し、一箇所を修正します。

## 到達チェックと再挑戦

別の人が再現できる手順、途中状態、終了、通常・境界・空の結果を示せれば到達です。ずれた場合はカード3枚に減らして一比較ずつ記録し、期待結果と一致してから元の課題へ再挑戦します。
