# 比較と条件分岐

## 学習目標

1. 値を保存する代入と、二つの値を比べる比較を区別できる。（C3.O1）
2. 与えられた値に対して、どの分岐が実行されるかをたどれる。（C3.O2）
3. 明示された規則を、単純な条件分岐として書き、境界を含めて試せる。（C3.O3）

## 前提知識

- C2で学んだ、変数、代入、上から順に実行される処理。
- 整数の大小と、二つの値が等しいこと。

## 導入: 規則に応じて次の処理を選ぶ

架空の星見高校メディアラボでは、印刷待ち行列へ依頼を追加する前に、次の規則を使います。

依頼と数値は教材用の合成データです。実在する生徒名、文書内容、利用履歴は扱いません。

> 必要枚数が20枚以下なら通常受付とし、20枚を超えるなら担当者確認とする。

条件によって次の処理を選ぶ考え方を**条件分岐**といいます。Pythonを書く前に、条件と二つの処理を言葉で分けます。

- 条件: 必要枚数は20枚以下か。
- 条件が成り立つとき: 通常受付にする。
- 条件が成り立たないとき: 担当者確認にする。

## 比較の結果は真か偽

比較は問いを作り、その結果は「成り立つ」か「成り立たない」のどちらかになります。Pythonでは、それぞれ `True` と `False` で表します。これらを**ブール値**といいます。

| 比較 | 意味 | 例 | 結果 |
| --- | --- | --- | --- |
| `<=` | 以下 | `12 <= 20` | `True` |
| `>` | より大きい | `12 > 20` | `False` |
| `==` | 等しい | `"mono" == "mono"` | `True` |
| `!=` | 等しくない | `"mono" != "color"` | `True` |

## `=` と `==` を区別する

C2では、`=` を使って変数へ値を保存しました。このレッスンで初めて扱う `==` は、左右の値が等しいかを調べます。

```python
pages = 12
print(pages == 12)
```

- `pages = 12`: `pages` に `12` を**代入する操作**。
- `pages == 12`: `pages` の値と `12` が等しいかを**比較する問い**。

`=` は状態を変えます。`==` は値を比べ、`True` または `False` を作ります。役割が異なるため、記号の数だけでなく、文全体が何をしようとしているかを考えます。

## 条件分岐の流れ

条件分岐は、まず条件を評価し、その結果に対応する一方の枝だけを実行します。

```text
必要枚数を求める
もし必要枚数が20枚以下なら
    通常受付と表示する
そうでなければ
    担当者確認と表示する
```

Pythonでは `if` と `else` を使って次のように書けます。

```python
pages = 3
copies = 4
sheets = pages * copies

if sheets <= 20:
    status = "通常受付"
else:
    status = "担当者確認"

print(sheets)
print(status)
```

出力は次のとおりです。

```text
12
通常受付
```

`if` の行の末尾には `:` を書きます。分岐の中で実行する行は、同じ幅で字下げします。字下げは見た目の飾りではなく、どの処理が枝に含まれるかを示します。

## 例題: 分岐をたどる

このモデルでは `ページ数 × 部数` を必要枚数とし、両面印刷は扱いません。次の二つの依頼は、どちらの状態になるでしょうか。

| 依頼番号 | 1部のページ数 | 部数 | 必要枚数 |
| --- | ---: | ---: | ---: |
| `Q105` | 6 | 3 | 18 |
| `Q106` | 8 | 3 | 24 |

規則は「20以下なら通常受付」です。

1. `Q105`: `18 <= 20` は `True` なので、`if` の枝を実行し、通常受付になる。
2. `Q106`: `24 <= 20` は `False` なので、`else` の枝を実行し、担当者確認になる。

20は規則が切り替わる**境界**です。19、20、21のように境界の直前、境界そのもの、直後を試すと、`<` と `<=` の取り違えを見つけやすくなります。

## 等しいかを使う分岐

印刷方法が `color` と等しいときだけ、確認メッセージを変える例です。

```python
print_mode = "color"

if print_mode == "color":
    message = "カラー設定を確認してください"
else:
    message = "モノクロで受け付けます"

print(message)
```

ここでも、`print_mode = "color"` は値の保存、`print_mode == "color"` は比較です。

## よくある間違い

- 比較したい場所に `=` を書く。値を保存するのか、等しいかを問うのかを先に言葉にします。
- `20枚以下` を `sheets < 20` と書く。20枚を含むので `<=` を使います。
- `if` と `else` の両方が実行されると考える。単純な二分岐では、条件に対応する一方だけが実行されます。
- `:` を忘れる、または字下げをそろえない。枝に含まれる行を同じ幅で字下げします。
- 通常の値だけを試す。規則が切り替わる境界と、その前後を試します。

## 自己確認

1. `copies = 2` と `copies == 2` は、それぞれ何をしますか。
2. `sheets` が20のとき、`sheets <= 20` の結果と実行される枝を説明できますか。
3. 「10ページ未満なら簡易確認、それ以外なら通常確認」という規則を、条件と二つの処理に分けられますか。

## 自己確認の診断と修正

確認できた状態は、比較を先に評価し、その結果から実行される枝を一つ選び、境界値で規則を確かめられることです。

- `=` と `==` が混ざる場合は、各行の横に「保存」または「比較」と書きます。
- 枝を選べない場合は、条件式だけを取り出して `True` か `False` を先に求めます。
- 境界が不明な場合は、規則に「以下」「未満」「以上」「より大きい」のどの語があるか確認します。

## まとめ

- 比較は二つの値の関係を調べ、`True` または `False` を作ります。
- `=` は代入、`==` は等しいかを調べる比較であり、役割が異なります。
- 条件分岐では条件を評価し、結果に対応する一方の枝を実行します。
- 境界の前、境界、境界の後を試すと、規則の書き間違いを見つけやすくなります。
