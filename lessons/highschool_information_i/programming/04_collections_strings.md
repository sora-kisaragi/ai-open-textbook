# コレクションと文字列

## 学習目標

1. 関係する複数の値を、別々の変数ではなく順序付きの値の集まりとして表現できる。（C5.O1）
2. 添字の範囲に注意しながら、要素へアクセスし、順に処理できる。（C5.O2）
3. 文章または表形式の値に合う、単純な表現を選べる。（C5.O3）

## 前提知識

- C2で学んだ、変数、代入、値の更新。
- C4で学んだ、固定回数の繰り返しと、繰り返しごとの変数の変化。

## 導入: 関係する値を一つの並びにする

架空の星見高校メディアラボで、待機中の印刷依頼3件のページ数を記録します。

```python
pages_q201 = 3
pages_q202 = 8
pages_q203 = 2
```

値が増えるたびに変数を追加すると、件数を数えたり、同じ処理を順に行ったりしにくくなります。順序のある複数の値をまとめたものを、ここでは**順序付きの値の集まり**または**並び**と呼びます。

Pythonの**リスト**や**文字列**は、並びとして扱える表現です。「列」は、表の縦方向の項目だけに使います。

この教材の依頼番号、ファイル名、ページ数はすべて架空の合成データです。

## リストで関係する値を表す

3件のページ数をリストにすると、次のように書けます。

```python
job_pages = [3, 8, 2]
print(job_pages)
```

角括弧の中に並ぶ一つ一つの値を**要素**といいます。リストにすると、「待機中の依頼のページ数」という関係を一つの変数で表せます。

## 添字で位置を指定する

並びの要素には位置があります。Pythonでは、先頭の添字は0です。

| 要素 | 3 | 8 | 2 |
| --- | ---: | ---: | ---: |
| 添字 | 0 | 1 | 2 |

```python
job_pages = [3, 8, 2]

print(job_pages[0])
print(job_pages[2])
```

出力は次のとおりです。

```text
3
2
```

要素が3個なら、有効な添字は0、1、2です。`job_pages[3]` は4番目の要素を求めますが、その要素はないためエラーになります。要素数と最後の添字を区別します。

## リストの要素を順に処理する

すべての要素に同じ処理を行うときは、C4で学んだ繰り返しを使えます。

```python
job_pages = [3, 8, 2]
total_pages = 0

for pages in job_pages:
    total_pages = total_pages + pages
    print("追加:", pages, "合計:", total_pages)

print("総ページ数:", total_pages)
```

出力は次のとおりです。

```text
追加: 3 合計: 3
追加: 8 合計: 11
追加: 2 合計: 13
総ページ数: 13
```

この書き方では、`pages` に要素が3、8、2の順に入ります。添字を自分で増やさないため、すべての要素を順に処理する目的に合います。

## 文字列も文字の並び

文字列は、文字が順に並んだものとして扱えます。

```python
job_code = "Q203"

print(job_code[0])
print(job_code[1])
```

出力は次のとおりです。

```text
Q
2
```

`job_code[0]` は先頭の文字 `Q`、`job_code[1]` は2番目の文字 `2` です。見た目が数字でも、文字列の中の `"2"` は文字です。数値 `2` と同じ扱いではありません。

## 目的に合う表現を選ぶ

すべてを同じ形で表す必要はありません。

| 表したいもの | 適した単純な表現 | 理由 |
| --- | --- | --- |
| 一つの依頼のファイル名 `festival_map.png` | 文字列 | 文字の順序を保つ一つのテキストだから |
| 待機中3件のページ数 `3, 8, 2` | リスト | 同じ種類の値の並びを順に処理したいから |
| 複数依頼の番号、ページ数、印刷方法 | 行と列を持つ表 | 依頼ごとに複数の項目を対応付けたいから |

表形式の合成データを考えます。

| 依頼番号 | ページ数 | 印刷方法 |
| --- | ---: | --- |
| `Q201` | 3 | `mono` |
| `Q202` | 8 | `color` |
| `Q203` | 2 | `mono` |

各行は一つの依頼、各列は同じ意味の項目です。Pythonで小さく表す方法の一つは、各行をリストにし、その行をさらにリストへ並べることです。

```python
jobs = [
    ["Q201", 3, "mono"],
    ["Q202", 8, "color"],
    ["Q203", 2, "mono"],
]

for job in jobs:
    print(job[0], job[1], job[2])
```

この例では、各行で0番目が依頼番号、1番目がページ数、2番目が印刷方法です。項目が増えた複雑なデータでは、意味の分かる列名を持つ表や別のデータ構造が適することもあります。ここでは、値の並びと表の関係を確かめるための単純な表現として使います。

## 例題: 必要な表現を選び、処理する

次の目的に合う表現を考えます。

> 待機中の依頼番号を順に表示し、件数を確認したい。

依頼番号は複数あり、順番に同じ処理をするため、文字列を要素に持つリストが適します。

```python
job_codes = ["Q201", "Q202", "Q203"]

for job_code in job_codes:
    print("待機中:", job_code)

print("件数:", len(job_codes))
```

`len(job_codes)` は要素数の3を返します。最後の有効な添字は2なので、要素数とは1ずれます。

## よくある間違い

- 関係する値を多数の別変数へ分ける。順に処理する同じ種類の値なら、リストへまとめられないか考えます。
- 先頭の添字を1と考える。Pythonのリストや文字列では先頭が0です。
- 要素数3のリストで添字3を使う。有効な添字は0から2までです。
- 数値 `2` と文字 `"2"` を同じものとして扱う。計算する数値か、並びを保つ文字かを確認します。
- 一つの文章を文字ごとのリストに分けないと処理できないと考える。文字列自体を文字の並びとして扱えます。
- 表の列の意味を決めず、位置だけで値を並べる。各位置が何を表すかを表や説明で明示します。

## 自己確認

1. `[3, 8, 2]` の要素数、先頭の添字、最後の添字をそれぞれ答えられますか。
2. 文字列 `"Q203"` の添字2にある文字は何ですか。
3. ファイル名、複数のページ数、複数依頼の表には、それぞれどの表現が適しますか。理由も説明できますか。

## 自己確認の診断と修正

確認できた状態は、データ同士の関係から表現を選び、添字の範囲を示し、すべての要素を繰り返しで処理できることです。

- 添字がずれる場合は、要素の下に0から番号を書きます。
- リストと表を選べない場合は、「値は同じ種類か」「一件に複数の項目があるか」を確認します。
- 繰り返しで要素を飛ばす場合は、`for 要素 in リスト` の形で直接たどれるか考えます。

## まとめ

- リストは、関係する複数の値を順序付きの集まりとして表せます。
- Pythonの添字は0から始まり、最後の添字は要素数より1小さくなります。
- 文字列も文字の並びであり、位置を指定したり順に処理したりできます。
- 一つのテキスト、同じ種類の値の並び、複数項目を持つ表を、目的に応じて使い分けます。
