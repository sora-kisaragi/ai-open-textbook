# Contributing

Thank you for contributing to AI Open Textbook Starter.

## Contribution types

- Lessons
- Exercises
- Answers
- Rubrics
- Teacher guides
- Review findings
- Accessibility improvements
- Validation scripts
- Data schema improvements

## Workflow

This project follows **GitHub Flow**. See `docs/GIT_WORKFLOW.md` for the full rules
(branch naming, commit conventions, merge strategy, and releases).

1. Open or claim an issue.
2. Branch from the latest `main` using `<type>/<issue-number>-<short-description>`.
3. Modify content and data together.
4. Add a revision record.
5. Commit using Conventional Commits.
6. Run validation.
7. Open a pull request linked to the issue.
8. Request human review.
9. Squash merge after approval and delete the branch.

Never push directly to `main`.

## Review standard

A PR is not ready until it answers:

- What learning objective does this change support?
- What data records changed?
- What tests were run?
- What still needs human review?
- What sources or evidence support the content?

## AI usage

AI assistance is allowed, but AI output must be reviewed.
Do not publish AI-generated material without human approval.
