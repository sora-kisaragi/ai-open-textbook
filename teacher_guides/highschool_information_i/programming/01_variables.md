# 教師用ガイド: 変数と代入

## レッスンのねらい

学習者が、代入を数学の等式ではなく、状態を更新する操作として理解することを目指します。
このレッスンでは、比較としての等しいことや `==` は主題にしません。

## 前提確認

プログラムが上から順に実行されることは前提にせず、授業のはじめの診断とミニレッスンで教え、確認します。
足し算そのものは前提にせず、`x = x + 2` を扱う直前に短く確認します。

## 進め方の例

1. 三つの自然言語命令を上から読み、各命令後の箱の値を表へ書く。
2. 順序を飛ばした誤答を診断し、「上から一行ずつ」の規則を確認する。
3. ゲームの得点はどこに保存されていると思うかを学習者に問いかける。
4. 変数を「値に付けた名前」として導入し、Pythonでは名前を値に対応付けることを確認する。「箱」「保存」は初学者向けの比喩であり、固定した物理的な保存場所という意味ではない。
5. `score = 80` と `print(score)` を示し、保存と表示を区別する。
6. `score = score + 10` を示し、代入が変数の値を更新する操作であることを話し合う。
7. 印刷待ちページ数を表す `pages_waiting` へ転移し、名前と状態の対応を確認する。
8. 練習を誘導・基礎、標準、統合の順で使う。

教材中の依頼IDと数値は合成データです。実在する生徒名、文書内容、印刷履歴、支援情報を例や提出物へ使わせません。

## 時間配分の目安

学習者の予測や対話に応じて、次の時間帯を組み替えて使います。固定された授業時間ではありません。

- 導入と変数の説明: 5〜10分程度
- 実行順序の診断とミニレッスン: 5〜10分程度
- `score = 80` と `print(score)` の確認: 5〜10分程度
- `x = x + 2` による値の更新: 5〜10分程度
- 自己確認と練習問題: 10〜20分程度

## 想定される誤解

- 学習者は `=` を数学の等号として読むことがある。
- 学習者は変数名そのものが値だと考えることがある。
- 学習者は実行順序に気づかないことがある。
- 学習者は `print` が値を保存する命令だと誤解することがある。

## 形成的評価

コードを実行する前に、表示される出力を学習者に予測させます。
誤答が出た場合は、各行の実行後に変数へ保存されている値を一つずつ確認します。

## 指導上の注意

練習は次の順に使えます。表示では日本語ラベルを先に示し、IDは参照用として添えます。

### 誘導・基礎

- **値を変数に保存する**（`prob.info1.variables.001.v1`）— 教師または本文の支援付き
- **代入後の出力を予測する**（`prob.info1.variables.002.v1`）— 教師または本文の支援付き
- **名前を付けて値を保存する利点を説明する**（`prob.info1.variables.003.v1`）
- **目的が伝わる変数名を選ぶ**（`prob.info1.variables.004.v1`）

`.001` と `.002` は形成的な誘導練習であり、独立したカバレッジ証拠として扱いません。
`.003`〜`.008` は、解答支援を外した独立評価として実施し、到達度の証拠にできます。

### 標準

- **印刷待ちページ数を更新する**（`prob.info1.variables.005.v1`）
- **複数回の代入後の出力を予測する**（`prob.info1.variables.006.v1`）
- **各行の実行後の値をたどる**（`prob.info1.variables.007.v1`）

### 統合

- **印刷待ちの状態を変数で表す**（`prob.info1.variables.008.v1`）
`==` は比較を扱う後続の学習で導入し、このレッスンでは `=` による代入に集中させます。
板書や口頭説明では、色だけに頼らず、変数名・代入する値・表示される値を言葉でも示します。
この内容は人間レビューが終わるまで承認済みまたは公開済みとして扱いません。
