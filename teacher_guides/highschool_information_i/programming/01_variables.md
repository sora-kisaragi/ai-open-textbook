# 教師用ガイド: 変数と代入

## レッスンのねらい

学習者が、代入を数学の等式ではなく、状態を更新する操作として理解することを目指します。
このレッスンでは、比較としての等しいことや `==` は主題にしません。

## 前提確認

授業のはじめに、プログラムが上から順に実行されることを確認します。
足し算そのものは前提にせず、`x = x + 2` を扱う直前に短く確認します。

## 進め方の例

1. ゲームの得点はどこに保存されていると思うかを学習者に問いかける。
2. 変数を、名前の付いた保存場所として導入する。
3. `score = 80` を示し、変数 `score` に値 `80` が保存されることを確認する。
4. `print(score)` を示し、保存された値を画面で確認する方法として扱う。
5. `score = score + 10` を示し、代入が変数の値を更新する操作であることを話し合う。
6. 短い値の予測問題に取り組ませる。

## 時間配分の例

- 導入と変数の説明: 5分
- `score = 80` と `print(score)` の確認: 7分
- `x = x + 2` による値の更新: 8分
- 自己確認と練習問題: 10分

## 想定される誤解

- 学習者は `=` を数学の等号として読むことがある。
- 学習者は変数名そのものが値だと考えることがある。
- 学習者は実行順序に気づかないことがある。
- 学習者は `print` が値を保存する命令だと誤解することがある。

## 形成的評価

コードを実行する前に、表示される出力を学習者に予測させます。
誤答が出た場合は、各行の実行後に変数へ保存されている値を一つずつ確認します。

## 指導上の注意

練習では `prob.info1.variables.001.v1` と `prob.info1.variables.002.v1` を使います。
`==` は比較を扱う後続の学習で導入し、このレッスンでは `=` による代入に集中させます。
板書や口頭説明では、色だけに頼らず、変数名・代入する値・表示される値を言葉でも示します。
この内容は人間レビューが終わるまで承認済みまたは公開済みとして扱いません。
