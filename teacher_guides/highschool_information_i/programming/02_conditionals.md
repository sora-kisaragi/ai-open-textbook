# 教師用ガイド: 比較と条件分岐

## レッスンのねらい

真偽を持つ問いを一つずつ評価し、`and`、`or`、`not` の意味で組み合わせてから、比較結果に対応する枝をたどります。概念を自然言語と真理値表で確認した後、代入 `=`、比較 `==`、複合条件をPythonへ対応付けます。

教材中の依頼ID、枚数、受付結果は合成データです。実在する生徒名、文書内容、印刷履歴を例や提出物へ使わせません。

## 前提確認

C2の変数、代入、更新、上から順に実行される処理を確認します。`score = 80` の役割を「値を保存する」と説明できれば十分です。比較記号 `==` はC2で先取りせず、ここで新しい役割として導入します。

## 学習者用本文との対応

1. 「20ページ以下なら通常受付」という規則を、条件、成り立つとき、成り立たないときへ分ける。
2. 比較が `True` または `False` を作ることを確かめる。
3. 二つの問いを `and`、`or`、`not` で組み合わせ、Pythonなしで真偽を求める。（C3.O1）
4. `=` と `==` を、それぞれ「保存」と「比較」として区別する。
5. `if` / `else` の一方だけが実行されることを追跡する。（C3.O2）
6. 20ページを境界として19、20、21を試す。（C3.O3）

## 時間配分の目安

必修2時限、合計100分の配分です。学習者のプログラミング経験に応じ、構文説明よりも規則の言語化、追跡、境界テストへ時間を移す場合も、各時限50分に収めます。

- 規則を三要素へ分ける導入: 10分
- 比較、ブール値、`and` / `or` / `not`: 20分
- `=` と `==` の区別: 10分
- 分岐の追跡と字下げ: 15分
- Pythonでの実装と境界テスト: 30分
- 自己確認と練習への接続: 15分

2時限に分ける場合は、第1時を比較と追跡、第2時を実装とテストにします。時間帯は固定せず、追跡表で誤解が残る場合は実装開始を後ろへずらします。

## 進め方の例

1. 自然言語の規則から、条件と二つの処理を付箋または表へ分ける。
2. `12 <= 20` など、値が決まった比較を先に評価する。
3. AとBの真偽カードを使い、A and B、A or B、not Aを全組合せで評価する。
4. `pages = 12` と `pages == 12` を並べ、「状態を変えるか」「問いに答えるか」で分類する。
5. Pythonコードを隠した状態で、本文の18枚と24枚の枝を予測させる。
6. 予測後にコードを実行し、字下げが枝の範囲を示すことを確認する。
7. 19、20、21のテスト結果を表にし、規則と一致するか判断する。

## 想定される誤解

- `=` と `==` を記号の見た目だけで覚え、役割を説明できない。
- 比較が表示命令や分岐そのものだと考える。
- `and` と `or` を交換しても同じ結果になると考える。
- `not` がどの問いを否定しているかを確認しない。
- `if` と `else` の両方が順に実行されると考える。
- 「以下」と「未満」、「以上」と「より大きい」を混同する。
- `:` や字下げを装飾と考える。
- 一つの通常値が通れば規則全体が正しいと判断する。
- 最初から複数条件や入れ子の分岐へ進み、単純な枝を追跡できなくなる。

## 形成的評価

C3.O1は、自然言語の二つの問いから `and`、`or`、`not` をPythonなしで評価させ、その後で代入行と比較式の役割を説明させます。
C3.O2は、条件式を先に `True` / `False` とし、その後に実行する枝と最終状態を答えさせます。
C3.O3は、自然言語の規則を二分岐へ変換し、境界の前・境界・直後の期待結果と実行結果を比べさせます。

誤答時はコード全体をすぐ修正せず、条件式、枝、出力を別々に確認します。

## Python実行時の注意

学習者用コードは外部入力を使わないため、そのまま実行できます。`pages` と `copies` を変え、合計20と21を必ず試します。
エラーが出た場合は、`:`、字下げ、引用符、`=` / `==` の順に確認します。
`and`、`or`、`not` の単純な複合条件は中核です。`elif` と入れ子の分岐は中核に追加しません。

## アクセシビリティと包摂への配慮

分岐図だけで説明せず、「条件を評価する」「成り立つ枝を実行する」「合流する」という順序付き文章も示します。字下げは空白幅だけに頼らず、枝の開始と終了を読み上げます。正誤を赤と緑だけで示さず、`True` / `False` と日本語の説明を併記します。

## 合成データと出典上の注意

星見高校、印刷依頼、20ページの受付規則は架空です。実在する学校の印刷方針や運用上の推奨を示すものではありません。規則は分岐と境界テストを学ぶための単純化です。
