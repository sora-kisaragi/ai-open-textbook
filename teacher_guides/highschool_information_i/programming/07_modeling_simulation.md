# 教師用ガイド: モデル化とシミュレーション

## レッスンのねらい

固定された印刷依頼を使い、待ち行列モデルの変数、パラメータ、関係、仮定、範囲、除外を明示します。同じ一件を手作業とPythonでたどり、速度だけを変える感度分析から、結果の有用性と限界を評価します。

- C8.O1: モデルの構成要素と範囲を特定する。
- C8.O2: 同じ規則を手計算とPythonで実行し、結果を照合する。
- C8.O3: パラメータ変更の影響、仮定、限界、改善を評価する。

## 前提確認

A5の「モデルは現実の一部を目的に合わせて選ぶ表現」であることを確認します。条件分岐、繰り返し、関数に加え、整数の割り算と切り上げは本文の例を使って短く確認します。

## 学習者用本文に沿った進め方

1. 「昼休みの待ち時間を調べる」という目的を示し、必要な変数と不要な詳細を分ける。
2. 変数、パラメータ、関係、仮定、範囲、除外を表へ整理する。
3. `M401` と `M402` を手作業でたどり、開始、終了、待ち時間を求める。
4. リスト行、単純な分岐、ループ、カウンタ、関数だけのPythonを実行し、手計算と同じ規則か確認する。
5. 速度だけを2、4、8枚/分に変え、速度2の待ち時間 `[0, 3, 6, 7]` と平均4.0を含めて比較する。
6. 結果から言えること、言えないことを分類する。
7. 目的に必要な改善を一つ選び、追加する仮定と再テストを設計する。

## 時間配分の目安

理解確認に応じて組み替える時間帯です。中核は100〜130分、感度分析と評価を深める場合は150分程度までを想定します。

- モデルの目的、構成要素、範囲: 20〜30分
- 手作業のシミュレーション: 15〜25分
- Python実装の対応確認: 25〜35分
- 速度を変える感度分析: 20〜30分
- 結果、仮定、限界、改善の評価: 20〜35分

短縮時は感度分析を速度の二条件に絞れますが、仮定と限界の記述は残します。追加時間では、切替時間を一つのパラメータとして加え、速度変更と混同しない比較を行います。

## 形成的評価

- C8.O1: 項目を「入力データ」「変更するパラメータ」「固定した仮定」「除外」に分類させる。
- C8.O2: `M402` の開始時刻が到着時刻だけでは決まらない理由を説明させる。
- C8.O3: 感度分析で固定した条件、変えた条件、変化した結果を一文ずつ書かせる。

「速度が上がると待ち時間が減った」という観察と、「実機でも同じ割合で減る」という未確認の主張を区別できるか確認します。

## 想定される誤解

- シミュレーションを現実の実験そのものとみなす。
- パラメータと結果の変数を区別しない。
- 複数条件を一度に変えて原因を断定する。
- 精密な小数が表示されれば、モデルも正確だと考える。
- 速度2の待ち時間の合計16を件数4で割らず、平均を誤る。
- 除外事項を失敗として隠す、または詳細を増やすほど必ずよいモデルになると考える。

## 支援と発展

時刻を数直線、表、番号付き文章のいずれでも表せるようにします。計算負担が大きい場合は途中値を記入した表を提供し、モデルの判断へ集中させます。

学習者必須コードには例外、`max`、辞書構築、`len`、`sum`、生成式を使いません。発展では、切替時間または最大枚数のどちらか一つだけを追加し、追加前後の結果とモデルの複雑さを比べます。

## データ、倫理、出典に関する注意

学校名、メディアラボ、依頼ID、到着時刻、枚数、速度はすべて教材用の架空設定と合成データです。実在する生徒の利用記録や文書内容を収集させません。この結果は実在機器の性能、運用の安全性、導入効果を証明しません。

## 練習問題

- **印刷待ちモデルの変数と仮定を分類する**（`prob.info1.programming.modeling.simulation.001.v1`、C8.O1）
- **手計算とPythonのシミュレーションを照合する**（`prob.info1.programming.modeling.simulation.002.v1`、C8.O2）
- **処理速度を変えて感度を調べる**（`prob.info1.programming.modeling.simulation.003.v1`、C8.O3）
- **結果の使える範囲と改善案を評価する**（`prob.info1.programming.modeling.simulation.004.v1`、C8.O1〜O3）
