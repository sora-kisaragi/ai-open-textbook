# 教師用ガイド: インターネット、Web、情報サービス

## レッスンのねらい

インターネット、Web、情報サービスを区別し、名前解決から画面提示までを単純化したサービスフローとして追跡します。障害箇所の診断と、構成・運用を含むサービス評価を接続します。

## 前提確認

D1の端末、経路、中継、規則を、送信元から宛先まで一文で説明させます。

## 学習者用本文との対応

1. 基盤、Web情報空間、利用目的を満たすサービスを分類する。（D2.O1）
2. 名前解決、接続、要求、処理、応答、提示をたどる。（D2.O2）
3. 観察証拠から障害候補を絞る。
4. 有効性、利用可能性、性能、アクセシビリティ、運用、安全性で評価する。（D2.O3）

## 時間配分の目安

70〜110分です。前半で用語と流れ、後半で障害切り分けとトレードオフ評価を扱います。

## 進め方と形成的評価

サービスフローのカードを並べ替え、各段階で観察できる証拠を添えさせます。D2.O1は分類理由、D2.O2は欠けた段階の補完、D2.O3は利用者要件と二つ以上の品質指標のトレードオフで確認します。

## 想定される誤解

- Webとインターネットを同義とする。
- サービスを画面だけとする。
- 失敗原因をすべてネットワークにする。
- 速度だけを品質とする。

## 支援とアクセシビリティ

インターネット、Web、サービスの用語表と、六段階の番号付きフローカードを用意します。図を使わず文章だけで順序や障害候補を説明する回答も認めます。長い記述が負担になる場合は、段階、観察証拠、候補原因を三列の表で選択・記入できるようにします。

## 合成データと出典上の注意

`.test` の名前と図書サービスは架空です。W3C Web ArchitectureはWebの一般的な識別・相互作用概念を確認する資料として使い、教材の六段階フローや評価項目を規定するものとはしません。

## 練習問題

- **インターネット、Web、サービスを区別する**（`prob.info1.networks.internet.web.001.v1`）: D2.O1
- **検索要求の流れをたどる**（`prob.info1.networks.internet.web.002.v1`）: D2.O2
- **障害箇所を証拠から切り分ける**（`prob.info1.networks.internet.web.003.v1`）: D2.O3
- **情報サービスを要件から評価する**（`prob.info1.networks.internet.web.004.v1`）: D2.O1〜O3
